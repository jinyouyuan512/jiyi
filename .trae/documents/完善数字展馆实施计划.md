我已完成对“数字展馆”模块的现状调研。目前前端页面 `Museum.tsx` 结构完整但使用静态数据，后端已具备完善的 `attractions`（景点/博物馆）和 `artifacts`（文物）数据表及 tRPC 接口。

基于此，我制定了以下完善计划，重点在于**数据对接**和**交互体验升级**：

## 1. 数据层对接 (Frontend Integration)
- [ ] **接入后端 API**：在 `Museum.tsx` 中引入 `trpc` 客户端。
- [ ] **替换 VR 体验数据**：调用 `trpc.attractions.list` 接口，筛选 `category` 为 `museum` 或 `memorial` 的数据，替换现有的静态 `exhibits` 列表。
- [ ] **替换文物数据**：调用 `trpc.artifacts.list` 接口，获取真实文物数据替换静态 `artifacts` 列表。
- [ ] **状态管理**：添加数据加载中（Skeleton 骨架屏）和加载失败的 UI 处理。

## 2. 交互与功能增强 (Interaction & Features)
- [ ] **搜索功能实现**：连接后端的 `search` 接口，使“口述历史”和“文物档案”部分的搜索框真正可用。
- [ ] **详情跳转**：
    - 点击 VR 展馆卡片跳转到 `AttractionDetail` 页面或直接打开 VR 链接（如果存在）。
    - 点击文物卡片展示文物详情（模态框或跳转）。
- [ ] **口述历史模块**：由于目前后端暂无专门的“口述历史”表，将暂时保留该部分的 UI 演示状态，或尝试复用 `artifacts` 接口（若有相关分类数据）。

## 3. 验证与测试
- [ ] **功能验证**：确保数据能正常加载，搜索能返回结果。
- [ ] **UI 检查**：确保在不同屏幕尺寸下（响应式）显示正常。

该计划将把“数字展馆”从一个静态演示页面转变为一个动态、可交互的功能模块。
