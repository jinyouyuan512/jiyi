我理解您的需求是将数字展馆升级为**基于虚幻引擎 (UE) 级别**的沉浸式体验平台。这将从单纯的网页浏览转变为**可交互的 3D 空间漫游**。

鉴于当前是 Web 环境，我制定了以下**混合架构方案**，既能模拟 UE 的高端接入流程，又能通过 Web3D 技术提供即时的沉浸体验：

### 1. 核心架构升级
*   **双模引擎支持**：
    *   **UE Pixel Streaming 模式**：预留云渲染接口，支持接入真实的 UE5 实时渲染流（通过 WebRTC/iframe）。
    *   **Web3D 原生模式**：引入 `Three.js` + `React Three Fiber`，构建轻量级 3D 场景作为默认/降级体验，确保无插件即刻访问。

### 2. 数据库与数据层扩展
*   **升级 `attractions` 表**：新增 `era` (时代背景)、`duration` (体验时长)、`streamUrl` (UE流地址) 字段。
*   **新增 `experience_records` 表**：记录用户的体验进度、开始/结束时间，用于生成证书。

### 3. 前端沉浸式组件开发
*   **3D 视口组件 (`ImmersiveViewport`)**：
    *   实现第一人称/轨道视角控制。
    *   集成**交互热点 (Hotspots)**：点击场景物体弹出历史详情。
*   **交互式导览 UI**：
    *   实时显示当前位置/历史解说字幕。
    *   任务/进度追踪系统。
*   **荣誉证书系统**：体验结束后，自动生成包含用户 ID 和体验内容的“红色文化传承证书”。

### 4. 实施步骤
1.  **环境准备**：安装 3D 图形库 (`three`, `@react-three/fiber` 等)。
2.  **数据迁移**：更新数据库结构并补充测试数据（添加体验时长、时代背景）。
3.  **核心开发**：
    *   开发 3D 播放器组件（支持 UE 流与 Web 模型切换）。
    *   实现场景内的交互逻辑（移动、触发事件）。
4.  **业务闭环**：实现“开始体验 -> 交互学习 -> 完成打卡 -> 获取证书”的完整流程。

该方案将完全满足您的验收标准，让用户真正“走进”历史。
